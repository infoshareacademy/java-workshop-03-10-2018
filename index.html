Ale jak to działa?

Rysować kolejne warstwy i HTTP

serwer

  • W klasie TaskList4000 dzieje się trochę magii, ale widać tam słowo "Tomcat"
  • Tomcat jest serwerem, który umie interpretować kod Javy
  • Magia sprawia, że Tomcat jest częścią budowanej przez nas aplikacji
  • Magia mówi, na jakie klasy i foldery Tomcat powinien patrzeć
  • Tomcat jest odpalony lokalnie, stąd adres "localhost:8080"

api/test

  • Gdy wpisujemy adres, np. "localhost:8080", przeglądarka dodaje od siebie "http://"
  • W internetach HTTP jest wszechobecne!
  • Najpopularniejszą metodą protokołu HTTP jest GET
  • Przeglądarka wysłała żądanie (request) HTTP GET pod adres, np. localhost:8080/api/test
  • Jedną z klas w Tomcat'cie jest WelcomeServlet (który rozszerza HttpServlet)
  • WelcomeServlet jest powiązany z adresem "/api/test"
  • WelcomeServlet ma metodę doGet, obsługującą żądanie HTTP GET i odpowiedź na to żądanie (response)
Tu nie tylko request-response ale i REST

doGet (1/2)

  • req.getParameter("name") to instrukcja, pobierająca z żądania parametr "name"
    • W przypadku http://localhost:8080/api/test?name=Mistrz jest to "Mistrz"
  • Gdy nie ma parametru name, jego wartość to null
  • Można pisać: 
    
                            String name = req.getParameter("name");
                            if(name == null) {
                                name = "world";
                            }
  • Ale można też: 
    
                            var name = Optional.ofNullable(req.getParameter("name"))
                              .orElse("world");

doGet (2/2)

  • Parametry są przesyłane jako String (tekst)
  • Integer.valueOf(text) pozwala zamienić text na liczbę całkowitą
  • Klasa WelcomeServlet ma prywatne pole typu WelcomeService
  • WelcomeService zwraca odpowiedni tekst, dla zadanych name oraz id
  • Umieszczenie tekstu w odpowiedzi (jeden ze sposobów): 
    resp.getWriter().write(fullText);

WelcomeService

  • Serwisy to miejsca w kodzie, gdzie trafia logika biznesowa ("mięcho") aplikacji
  • WelcomeService ma metodę, która "wie" jak dla podanej nazwy oraz id powitania, stworzyć tekst, witający użytkownika z daną nazwą
  • W pierwszym kroku metody następuje pobranie powitania z bazy danych
  • Do komunikacji z bazą, jest użyte narzędzie Hibernate
  • Pobrane powitanie jest "sklejane" z resztą wiadomości
  • Gdy obiekt powitania "skleja się" z tekstem, tak naprawdę wołana jest na nim metoda toString

baza danych (1/2)

  • Hibernate pozwala na łączenie się z bazą danych
  • Jest to tzw. ORM - mapowanie obiektowo-relacyjne
  • Plik resources/hibernate.cfg.xml mówi
    • Jaką mamy bazę, gdzie i jak się z nią łączyć
    • Co robić przy podłączaniu (u nas: create-drop)
    • Które obiekty łączą się z tabelami
  • Używamy bazy relacyjnej H2
  • Jest ona trzymana w pliku ~/task-list, ("~" to folder użytkownika systemu operacyjnego)
  • Operacje odczytu/zapisu do bazy danych, dzieją się w tzw. sesjach (sesja = trwa połączenie z bazą)

baza danych (2/2)

  • Klasa Welcome z pakietu model, reprezentuje tabelę "greetings" z bazy danych
  • Każde pole z tej klasy jest osobną kolumną w bazie danych
  • Każdy wpis w tabeli musi mieć swój unikalny identyfikator
  • W WelcomeService wyciągamy po id wiersz z tabeli związanej z klasą Welcome: 
    
                            HibernateUtil.getSessionFactory()
                              .openSession().get(Welcome.class, id);
  • Start Tomcata odpala DbInit: contextInitialized
    • Ta metoda zapisuje 2 obiekty do bazy
    • Inaczej byłoby pusto (skutek create-drop)

Podsumowanie działania

  1. Tomcat serwuje naszą aplikację na localhost:8080
  2. Przy starcie Tomcata następuje usunięcie danych z bazy H2 i wpisanie 2 przykładowych wartości
  3. Zapytanie np. o localhost:8080/api/test wywołuje odpowiednią metodę z WelcomeServlet
  4. WelcomeServlet woła WelcomeService, który wyciąga odpowiednie dane z bazy przez Hibernate'a
  5. Powstaje String, który trafia do przeglądarki przez HttpServletResponse

Nowy HttpServlet

  1. Metoda doGet
  2. Jak umieścić wszystkie Todo wewnątrz response'a?
  3. Zamiana obiektu w JSON!
    • ObjectMapper (pakiet do ściągnięcia: com.fasterxml.jackson.core.jackson-databind)

                            @WebServlet(name = "TodoServlet", urlPatterns = {"/api/todos/*"})
                            public class TodoServlet extends HttpServlet {
                                private final ObjectMapper objectMapper = new ObjectMapper();
                                private final TodoService service = new TodoService();

                                @Override
                                protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {
                                    resp.setContentType("application/json;charset=UTF-8");
                                    objectMapper.writeValue(resp.getOutputStream(), service.getAllTodos());
                                }

    • Stworzone Todo nie może być od razu zrobione
  2. Serwis powinien dodawać nowe Todo i zwracać stworzony obiekt, już z ID
  3. Zwracamy stworzony obiekt (jako JSON)

                            @Override
                            protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws IOException {
                                var text = req.getReader().lines().collect(Collectors.joining(System.lineSeparator()));
                                resp.setContentType("application/json;charset=UTF-8");
                                objectMapper.writeValue(resp.getOutputStream(), service.addTodo(text));
                            }

Serwis i dalej

                            public Todo addTodo(String todoText) {
                                var todo = new Todo(todoText);
                                var session = HibernateUtil.getSessionFactory().openSession();
                                var transaction = session.beginTransaction();
                                session.persist(todo);
                                transaction.commit();
                                session.close();
                                return todo;
                            }

Zadanie 3

Edycja zadań Todo

HttpServlet - zmiana istniejącego Todo

  1. Adres będzie zawierał ID zadania, które powinniśmy zmienić
  2. Np. localhost:8080/api/todos/2, gdy zostało kliknięte Todo o ID=2

                            @Override
                            protected void doPut(HttpServletRequest req, HttpServletResponse resp) throws IOException {
                                var pathInfo = req.getPathInfo(); // np. "/123" dla "localhost:8080/api/todos/123"
                                int id = Integer.valueOf(pathInfo.substring(1));
                                resp.setContentType("application/json;charset=UTF-8");
                                objectMapper.writeValue(resp.getOutputStream(), service.toggleTodo(id));
                            }

Serwis

Podmianka statusu w obrębie transakcji


                            public Todo toggleTodo(int id) {
                                var session = HibernateUtil.getSessionFactory().openSession();
                                var transaction = session.beginTransaction();
                                var todo = (Todo) session.get(Todo.class, id);
                                todo.setDone(!todo.isDone());
                                transaction.commit();
                                session.close();
                                return todo;
                            }
